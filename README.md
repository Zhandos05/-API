# Авторизация API (Java)

## Жоба мақсаты
Бұл жоба пайдаланушыны тіркеу және жүйеге кіру (авторизация) процесін көрсету үшін жасалған. Парольдер қауіпсіз түрде хэштеу арқылы сақталады.

---

## Функционал
- Пайдаланушыны тіркеу (Register)
- Жүйеге кіру (Login)
- Парольді хэштеу

---

## Хэштеу деген не?
Хэштеу — бұл парольді арнайы алгоритм арқылы өзгертіп, қайтымсыз кодқа айналдыру процесі. Яғни бастапқы парольді қайта қалпына келтіру мүмкін емес.

---

## Неге парольді ашық сақтауға болмайды?
Егер пароль ашық түрде сақталса, база бұзылған жағдайда барлық қолданушылардың парольдері ұрланады. Бұл үлкен қауіпсіздік мәселесіне әкеледі. Сондықтан пароль тек хэш түрінде сақталуы керек.

---

## Авторизация қалай жұмыс істейді?
1. Пайдаланушы тіркеледі (register)
2. Пароль хэшке айналып базаға сақталады
3. Кіру кезінде (login) енгізілген пароль қайта хэштеледі
4. Ол базадағы хэшпен салыстырылады
5. Егер сәйкес болса — жүйеге кіреді

---

## Қорытынды
Бұл жоба авторизацияның негізгі принциптерін түсіндіреді және қауіпсіздіктің маңызды екенін көрсетеді.
